Case Study 01 · Procore Technologies

Enterprise Agentic AI Platform — 0% to 90%+ Response Accuracy

GenAI / Agentic Production 3,000+ Customers
+

Problem

Procore's AI assistant was returning ~45% accurate responses — hallucinating project data, surfacing wrong documents, and failing on multi-domain queries. Enterprise customers were churning on AI features before they could demonstrate ROI. The platform had no grounding strategy and no evaluation framework to measure or improve.

Strategy & Execution

Designed a supervisor orchestration layer with intent-routing to domain-specific agents using real-time manifest resolution. Built just-in-time grounding workflows with schema-driven action layers. Established Evaluation-as-a-Service with transaction-level metrics (correctness, recall, completeness). Shipped multimodal RAG pipelines supporting text, document, and image intelligence. Defined AI governance: agent registries, versioning, ML exclusions, hybrid-cloud compliance across AWS, GCP, Azure.

90%+ Response accuracy (from 45%)
3,000+ Enterprise customers on platform
80% User productivity improvement

Tech Stack

GPT-4 / Claude / Gemini Elasticsearch Vector Embeddings MCP Protocol AWS / GCP / Azure SQS RAG Pipelines YOLO + OCR + BERT

Case Study 02 · Autodesk, Inc.

License-to-Subscription Migration — $1B ARR Platform

Platform / API $1B ARR Business Transformation
+

Problem

Autodesk was pivoting from perpetual licenses to a subscription model — one of the most complex commercial transformations a software company can undertake. The existing infrastructure had no transactional API layer to support subscription creation, entitlement management, or partner order placement at scale. Getting this wrong meant broken revenue recognition and customer trust damage.

Strategy & Execution

Led design of transactional subscription APIs — GET/POST microservices enabling partners to place orders, get pricing, manage products, and create subscription contracts. Defined full API contracts: schema, entities, filtering/sorting, error handling, rate limits (<250ms latency, 10K req/user). Built order-to-subscription sync: .com → SAP (order) → Salesforce (subscription with entitlements, product line items, start/end dates). Contributed to $1B ARR in FY19.

$1B ARR contribution FY19
25% YoY revenue growth via ML models
$1M+ Annual savings via NLP support bot

REST APIs SAP OMS Salesforce CPQ Microservices IBM Watson NLP Snowflake Databricks Spark / Flink

Case Study 03 · Procore Technologies

360° Executive Dashboard — Near-Real-Time Reporting at Scale

Data Platform Distributed Systems CAP Theorem Applied
+

Problem

Enterprise customers ($500M+ ARR accounts) were threatening non-renewal because Procore had no cross-domain reporting. Data was fragmented across Financials, PMQS, and Resource Management. Full materialization guaranteed consistency but took 45 minutes — customers needed data in under 5 minutes. The CAP tradeoff was real: strong consistency blocked data for 45 minutes; eventual consistency could deliver in 5 minutes with manageable staleness.

Strategy & Execution

Designed a partitioning strategy: top 50 enterprise customers received dedicated partitions for high-volume datasets; remaining customers shared a single optimized partition. Introduced incremental materialization across all domains. Audited DynamoDB → DocumentDB → MongoDB migration path to achieve <250ms P99 SLO on complex nested aggregations. Committed to full consistency within 10 minutes via incremental batch processing — a tradeoff customers explicitly approved.

5 min Near-real-time (from 45 min)
<250ms P99 query latency
$300M+ Revenue protected via retention

Tech Stack

MongoDB DynamoDB DocumentDB Incremental Materialization Partition Strategy Graph API Elasticsearch
